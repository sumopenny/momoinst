# MVSEP API 接口文档

## 1. 接口概览

## 2. 接口详情

### 2.1 创建分离任务

**接口名称**：创建分离任务
**接口路径**：`https://mvsep.com/api/separation/create`
**请求方法**：POST

#### 请求参数

| 参数名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| api_token | String | 是 | - | 您的 API Key |
| audiofile | Binary | 否 | - | 要处理的音频文件。如果设置了 url 参数或模型不使用文件输入，则不需要。对于 matchering 模型，使用 target_audiofile 和 reference_audiofile 作为输入，而不是 audiofile。 |
| url | String | 否 | - | 音频文件的链接。如果设置了 audiofile 参数，则不需要。 |
| remote_type | String | 否 | direct | 音频文件的远程类型。如果设置了 audiofile 参数，则不需要。支持的类型有 direct（直接下载）、mega（MEGA）、drive（Google Drive）、dropbox（Dropbox） |
| sep_type | Integer | 否 | 49 | 分离类型，默认为 49 (MVSep Karaoke (主唱，伴唱)) |
| add_opt1 | String | 否 | - | 分离类型的第一个额外字段 |
| add_opt2 | String | 否 | - | 分离类型的第二个额外字段 |
| add_opt3 | String | 否 | - | 分离类型的第三个额外字段 |
| output_format | Integer | 否 | 0 | 输出格式，默认为 0 (mp3 (320kbps))。支持的格式：0 - mp3 (320 kbps)，1 - wav (uncompressed, 16 bit)，2 - flac (lossless, 16 bit)，3 - m4a (lossy) |
| is_demo | Boolean | 否 | false | 是否将分离结果发布到示例页，默认为 false |

### 2.2 获取分离结果

**接口名称**：获取分离结果
**接口路径**：`https://mvsep.com/api/separation/get`
**请求方法**：GET

#### 请求参数

| 参数名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| hash | String | 是 | - | 此分离的哈希值 |
| mirror | Integer | 否 | 0 | 如果设置为 1，则使用另一个镜像，默认为 0。需要 api_token 和 1 个积分（如果设置为 1）。 |
| api_token | String | 否 | - | 用户的 API 密钥。如果 mirror 参数设置为 1，则为必需。 |

#### 请求示例

```bash
curl --location --request GET 'https://mvsep.com/api/separation/get?hash=20230327071601-0e3e5c6c85-13-dimensions.mp3'

# 使用镜像
curl --location --request GET 'https://mvsep.com/api/separation/get?hash=20230327071601-0e3e5c6c85-13-dimensions.mp3&mirror=1&api_token=YJbyIzPlBxCLG3puydDQRW8Es1f5sA'
```

#### 响应数据结构

| 字段名 | 数据类型 | 说明 |
|--------|----------|------|
| success | Boolean | false - 当文件哈希未找到、被删除或已过期<br>true - 当文件哈希值找到且有效 |
| status | String | not_found - 当作业无效时<br>waiting - 当作业处于队列中且尚未处理时<br>processing - 当作业正在处理时<br>done - 当作业已成功处理<br>failed - 当作业处理失败时<br>distributing - 当作业中的音频文件较大且正在分发到多个 GPU 实例时<br>merging - 当所有 Job 分配部分完成处理并合并时 |
| data | Object | 根据 status 键保存额外信息 |
| data.queue_count | Integer | 在用户的优先级中，显示未处理的作业计数（仅在 status 为 waiting 或 distributing 时显示） |
| data.current_order | Integer | 显示用户作业的顺序（仅在 status 为 waiting 或 distributing 时显示） |
| data.message | String | 显示状态描述（显示在所有状态上，当 status 为 failed 时显示错误原因） |
| data.algorithm | String | 显示作业中使用的算法（仅在 status 为 done 时显示） |
| data.algorithm_description | String | 显示作业中使用的算法详细信息（仅在 status 为 done 时显示） |
| data.output_format | String | 显示作业的输出格式（仅在 status 为 done 时显示） |
| data.tags | Object | 显示音频元标记（仅在 status 为 done 时显示） |
| data.input_file | Object | 显示输入音频下载详情（仅当 status 为 done 时显示） |
| data.input_file.url | String | 输入音频文件的下载链接 |
| data.input_file.size | String | 输入音频文件的大小 |
| data.files | Array | 显示输出音频下载详情（仅在 status 为 done 时显示） |

| data.files[].url | String | 输出文件下载链接 |

| data.finished_chunks | Integer | 显示大型作业的已完成部分的数量（仅在 status 为 distributing 时显示） |
| data.all_chunks | Integer | 显示大型作业的所有部分的编号（仅在 status 为 distributing 时显示） |

## 4. 错误码说明

| 错误码 | 描述 | 解决方案 |
|--------|------|----------|
| 401 | 未授权 - API 密钥无效 | 检查您的 API 密钥是否正确 |
| 400 | 请求参数错误 | 检查请求参数是否符合要求 |
| 404 | 资源未找到 | 检查哈希值是否正确，文件是否已删除或过期 |
| 429 | 请求过于频繁 | 减少请求频率，或升级到 Premium 计划 |
| 500 | 服务器内部错误 | 稍后重试，或联系客服 |

## 5. 接口调用注意事项

1. **API 密钥安全**：请妥善保管您的 API 密钥，不要在客户端代码或公共仓库中暴露。

2. **并发限制**：非 Premium 用户只能同时处理 1 个任务，Premium 用户可以处理无限个并发任务。

3. **文件大小限制**：请注意音频文件的大小限制，过大的文件可能会导致处理失败。

4. **响应处理**：对于获取分离结果的接口，需要根据 status 字段的值进行不同的处理：
   - `waiting`：任务在队列中，需要等待
   - `processing`：任务正在处理中
   - `done`：任务已完成，可以获取结果
   - `failed`：任务处理失败，需要查看错误信息
   - `distributing`：大型任务正在分发到多个 GPU 实例
   - `merging`：所有分配的部分正在合并

5. **镜像使用**：使用 mirror=1 参数会消耗 1 个积分，但可能会提供更快的下载速度。

6. **取消任务**：只有在任务尚未开始处理时，取消任务才能 refund 已使用的积分。

7. **队列查询**：使用 API 密钥查询队列信息时，可以获取当前计划的队列状态。

8. **输出格式**：默认输出格式为 mp3 (320kbps)，可以根据需要选择其他格式。

9. **示例页发布**：不需要将分离结果发布到示例页。

10. **远程文件类型**：当使用 url 参数时，可以指定 remote_type 来处理不同类型的远程存储服务。
